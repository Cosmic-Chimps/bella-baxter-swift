# Bella Baxter Swift SDK

Swift client for the Bella Baxter secrets management API, powered by
**[Apple's swift-openapi-generator](https://github.com/apple/swift-openapi-generator)**.

---

## Features

- 🍎 **Native Swift** — `async/await`, value types, zero Objective-C
- 🔧 **Build-time code generation** — OpenAPI client generated by SwiftPM at build time (no separate script)
- 📱 **All Apple platforms** — iOS 17+, macOS 14+, watchOS 10+, tvOS 17+
- 🔑 **Bearer auth** — API key injected on every request via middleware
- 📦 **Zero friction** — add the package, build, done

## Quick Start

```swift
import BellaBaxterSwift

// 1. Create client
let client = try BellaClient(
    BellaClientOptions(
        baseURL: URL(string: "https://api.example.com")!,
        apiKey: "bax-..."
    )
)

// 2. Pull all secrets for an environment (merges all providers)
let secrets = try await client.pullSecrets(
    projectRef: "my-app",       // project GUID or slug
    environmentSlug: "production"
)

// 3. Use them
let dbURL = secrets["DATABASE_URL"] ?? ""
let apiToken = secrets["THIRD_PARTY_TOKEN"] ?? ""
```

## API

### `BellaClientOptions`

| Parameter | Type | Description |
|-----------|------|-------------|
| `baseURL` | `URL` | Bella Baxter API base URL |
| `apiKey` | `String` | Bearer API key (`bax-...`) |
| `timeoutSeconds` | `TimeInterval` | Default: 30 |

### `BellaClient`

| Method | Description |
|--------|-------------|
| `listProjects(page:size:)` | List all accessible projects |
| `getProject(ref:)` | Get a project by GUID or slug |
| `listEnvironments(projectRef:)` | List environments in a project |
| `pullSecrets(projectRef:environmentSlug:)` | **Primary**: all secrets merged into `[String: String]` |
| `listSecrets(projectRef:environmentSlug:providerSlug:)` | Secrets for a specific provider |
| `exportSecretsAsEnv(projectRef:environmentSlug:providerSlug:)` | `.env`-formatted string |
| `injectIntoEnvironment(projectRef:environmentSlug:)` | Inject secrets into `ProcessInfo` (macOS/CLI) |

## iOS Integration Pattern

Store the client in your app's environment and load secrets at startup:

```swift
@main
struct MyApp: App {
    var body: some Scene {
        WindowGroup {
            ContentView()
                .task { await AppConfig.shared.loadFromBella() }
        }
    }
}

@MainActor
final class AppConfig {
    static let shared = AppConfig()

    private(set) var apiBaseURL = ""
    private(set) var featureFlags: [String: String] = [:]

    func loadFromBella() async {
        guard let url = URL(string: "https://api.example.com") else { return }
        let client = try? BellaClient(BellaClientOptions(baseURL: url, apiKey: "bax-..."))
        guard let secrets = try? await client?.pullSecrets(
            projectRef: "my-ios-app",
            environmentSlug: ProcessInfo.processInfo.environment["ENV"] ?? "production"
        ) else { return }

        apiBaseURL = secrets["API_BASE_URL"] ?? ""
        // map any other secrets you need
    }
}
```

## How Code Generation Works

`swift-openapi-generator` is a **SwiftPM build tool plugin**:

1. At `swift build` / Xcode build time, the plugin reads `Sources/BellaBaxterSwift/openapi.json`
   and `Sources/BellaBaxterSwift/openapi-generator-config.yaml`
2. It generates a type-safe `Client` struct with one method per API operation
3. `BellaClient.swift` wraps the generated `Client` with convenience methods + auth

To update the generated client when the API changes:

```bash
# From the repository root:
./apps/sdk/generate.sh          # local API running
./apps/sdk/generate.sh --dev    # dev/staging API
```

This copies the latest `openapi.json` into `Sources/BellaBaxterSwift/`, and SwiftPM
will regenerate the client on the next build automatically.

## Typed Secret Code Generation

`bella secrets generate swift` fetches the secrets manifest (key names + type hints, no values) from the Bella API and generates a typed `AppSecrets` struct. Each static property reads from `ProcessInfo.processInfo.environment` at runtime — no secret values are ever embedded in the generated file.

```bash
bella secrets generate swift \
  --project my-app \
  --environment production \
  --output Sources/AppSecrets/AppSecrets.swift
```

**Generated `AppSecrets.swift`:**

```swift
// Auto-generated by bella secrets generate swift — do not edit manually.
import Foundation

public struct AppSecrets {
    public static var databaseUrl: String {
        guard let v = ProcessInfo.processInfo.environment["DATABASE_URL"] else {
            fatalError("Secret 'DATABASE_URL' is not set.")
        }
        return v
    }

    public static var port: Int {
        guard let v = ProcessInfo.processInfo.environment["PORT"],
              let n = Int(v) else {
            fatalError("Secret 'PORT' is not set or is not a valid Int.")
        }
        return n
    }

    public static var enableFeatureX: Bool {
        guard let v = ProcessInfo.processInfo.environment["ENABLE_FEATURE_X"] else {
            fatalError("Secret 'ENABLE_FEATURE_X' is not set.")
        }
        return v == "1" || v.lowercased() == "true"
    }
}
```

### Usage

```swift
// Secrets must be in the process environment at launch.
// Use bella run, the Swift SDK, or inject via Xcode scheme environment variables.

let url  = AppSecrets.databaseUrl   // String — fatalError if missing
let port = AppSecrets.port          // Int — parsed automatically
```

Each property reads `ProcessInfo.processInfo.environment` on every access, so values injected before launch (e.g. by `bella run --`) are always current.

### Options

| Option | Default | Description |
|--------|---------|-------------|
| `-p, --project <slug>` | `.bella` context | Project slug |
| `-e, --environment <slug>` | `.bella` context | Environment slug |
| `--provider <slug>` | `default` | Provider slug |
| `-o, --output <path>` | `AppSecrets.swift` | Output file path |
| `--class-name <name>` | `AppSecrets` | Struct name |
| `--dry-run` | — | Print to stdout without writing |
